# 龟投论坛 — 后端接口文档 v1.0

> 基于前端功能全面梳理，覆盖用户系统、宠物系统、投票系统、对战系统、小游戏、赛季系统。

## 一、技术选型建议

| 层级 | 推荐方案 |
|------|----------|
| 语言/框架 | Node.js (Express/Koa) 或 Python (FastAPI) |
| 数据库 | PostgreSQL + Redis (缓存/排行榜) |
| 认证 | JWT Token |
| 实时通信 | WebSocket (投票/对战实时更新) |
| 定时任务 | node-cron / celery (每日登录结算、赛季结算) |
| AI 对话 | 接入通义千问 / DeepSeek API |

### 2.5 庄家赌局表 `duels`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| title | VARCHAR | 议题标题 |
| banker_id | UUID | 庄家（发起者） |
| banker_opinion | TEXT | 庄家立场 |
| challenger_opinion | TEXT | 对立面描述 |
| banker_stake | INTEGER | 庄家押注（即愿赔上限，冻结） |
| challenger_frozen_total | INTEGER | 挑战者冻结总额（下注额×95%，实时更新） |
| challenger_count | INTEGER | 挑战者人数 |
| entry_fee_total | INTEGER | 已收入场费总额（下注额×5%，已到庄家账上） |
| visibility | ENUM | public（公开，出现在广场）/ private（私人，仅邀请） |
| status | ENUM | open / sealed / pending_result / pending_confirm / disputing / settled / voided / cancelled |
| banker_wins | BOOLEAN | 庄家是否胜出（结算后填入） |
| banker_declared_at | TIMESTAMP | 庄家宣布结果的时间 |
| dispute_count | INTEGER | 异议数量 |
| banker_penalty | INTEGER | 庄家罚金（虚报/模糊时扣除） |
| invite_code | VARCHAR | 邀请码/链接（私人赌局必需） |
| settle_at | TIMESTAMP | 结算时间（必填，到时自动封盘） |
| created_at | TIMESTAMP | 创建时间 |

> 挑战者冻结总额 ≤ 庄家押注，满额时自动封盘。
> 公开赌局：入场费（5%）在挑战者加入时立即转入庄家余额，不参与冻结。
> 私人赌局：不收入场费，下注额100%冻结。
>
> **状态流转：**
> open → sealed（手动/满额/到settle_at自动封盘）→ pending_result（等庄家24h内宣布）
> → pending_confirm（庄家已宣布，挑战者24h内确认）→ 全部通过 → settled
> → 任一异议 → disputing（人工审核）→ settled / voided（作废+惩罚）
> 庄家24h超时未宣布 → 庄家直接判输 → settled
> settle_at到期无挑战者 → cancelled（庄家押注退还）

### 3.1 认证模块

#### `POST /api/auth/register`
注册新用户，赠送 1000 龟币。龟币为纯平台积分，仅通过系统行为产出/消耗，不可充值或转账。注册不赠送宠物（宠物系统在第三阶段）。

```
请求: { username: string, password: string }
响应: { user_id, token, turtle_coins: 1000 }
```

#### `POST /api/auth/login`
登录获取 token。

```
请求: { username: string, password: string }
响应: { user_id, token, profile: {...} }
```

#### `POST /api/auth/daily-login`
每日签到（北京时间判断），发放奖励 = 100 + 5 * login_streak。

```
请求: {} (需 Authorization header)
响应: {
  login_streak: number,
  reward: number,
  bonus_pet_reward: number,    // 龟种加成 (天使龟+40, 彩虹龟+60, 凤凰龟+100, 无头龟+50)
  dice_bonus: number|null,     // 骰子龟: 0~90 随机
  turtle_coins_after: number
}
```

### 3.5 投票模块

#### `GET /api/topics`
获取投票主题列表。支持热点榜和最新榜两种排序。

**热点榜算法**（调度任务每5分钟计算，结果存 Redis sorted set）:
```
heat = (投注人数×3 + 投注总额/100×2 + 浏览量×0.5) / (age_hours + 2) ^ 1.5
```
- 投注人数权重最高（参与度）
- 投注总额次之（信心强度）
- 浏览量辅助（防冷启动）
- 指数1.5：衰减比 HN(1.8) 慢，适合预测事件较长生命周期

```
查询: ?sort=hot|latest&category=all&status=active&page=1&limit=20
响应: { topics: [...], total, has_more }
```

#### `GET /api/topics/:id`
获取单个主题详情（含实时票数）。

```
响应: { id, title, yes_count, no_count, yes_odds, no_odds, status, expires_at }
```

#### `POST /api/topics/:id/vote`
投票下注。bet >= 100 时计入连胜。

```
请求: { choice: "yes"|"no", bet_amount: number }
响应: {
  success: true,
  turtle_coins_after,
  vote_streak_updated: boolean,
  fortune_bonus: number|null    // 财神龟: 首次投票+50
}
错误: INSUFFICIENT_FUNDS | ALREADY_VOTED | TOPIC_CLOSED
```

#### `POST /api/topics/:id/settle` (管理员)
结算投票，计算胜负和派奖。

```
请求: { result: "yes"|"no" }
响应: { winners_count, total_payout }
```

### 3.6 庄家赌局模块

#### `POST /api/duel/create`
做庄开局。庄家设定议题、立场、押注金额、公开/私人、结算时间（必填）。龟币即时冻结。

```
请求: {
  title, banker_opinion, challenger_opinion,
  banker_stake: number(100~10000),
  visibility: "public"|"private",
  settle_at: timestamp              // 必填，议题结果揭晓时间，到时自动封盘
}
响应: { duel_id, invite_code, duel: {...} }
错误: INSUFFICIENT_FUNDS | STAKE_OUT_OF_RANGE(100~10000) | SETTLE_TIME_INVALID | BANKER_BANNED
```

> 创建后议题、立场、押注、结算时间均**不可修改**。

#### `GET /api/duel/plaza`
赌局广场（仅公开赌局）。

```
查询: ?status=open|sealed|all&sort=hot|latest&page=1&limit=20
响应: { duels: [...], total, has_more }
```

#### `GET /api/duel/:id`
获取赌局详情。

```
响应: {
  duel: { title, banker_opinion, challenger_opinion, banker_stake, challenger_frozen_total, visibility, status, expires_at },
  banker: { user_id, username, stake, entry_fee_earned },
  challengers: [{ user_id, username, bet_amount, entry_fee, frozen_amount, joined_at }],
  remaining_capacity: number,        // 庄家押注 - 挑战者冻结总额 = 还能接多少（按冻结额算）
  my_role: "banker"|"challenger"|null
}
```

#### `POST /api/duel/:id/join`
挑战庄家。自动站对立面。公开赌局扣5%入场费给庄家+95%冻结；私人赌局100%冻结。

```
请求: { stake: number(50~5000) }
响应: {
  success,
  entry_fee: number,                 // 公开赌局: stake×5%（已付给庄家）；私人赌局: 0
  frozen_amount: number,             // 公开赌局: stake×95%；私人赌局: stake×100%
  turtle_coins_after,
  remaining_capacity
}
错误: INSUFFICIENT_FUNDS | STAKE_OUT_OF_RANGE | STAKE_EXCEEDS_CAPACITY | DUEL_SEALED | DUEL_EXPIRED | ALREADY_JOINED | CANNOT_CHALLENGE_SELF
```

> frozen_amount 不能超过剩余容量（庄家押注 - 已有挑战者冻结总额）

#### `POST /api/duel/:id/seal`
庄家手动封盘，停止接受新挑战者。也可到期/满额自动封盘。

```
响应: { success, challenger_frozen_total, challenger_count }
```

#### `POST /api/duel/:id/cancel`
取消赌局。仅 open 状态且**无任何挑战者**时可操作。赌局一旦有人加入则不可取消。

```
响应: { success, banker_refund: number }
错误: DUEL_HAS_CHALLENGERS | DUEL_NOT_OPEN
```

#### `POST /api/duel/:id/declare` (庄家)
庄家宣布结果，推送给每位挑战者。仅 settle_at 之后可操作。宣布后挑战者有24h确认窗口。

```
请求: { banker_wins: boolean }
响应: { success, confirm_deadline: timestamp }
错误: DUEL_NOT_READY | DECLARE_TIMEOUT
```

> 庄家需在 settle_at 后**24h内**宣布，超时则**庄家直接判输**，按庄家输结算。

#### `POST /api/duel/:id/confirm` (挑战者)
挑战者对庄家宣布的结果进行确认。24h内未操作视为自动通过。

```
请求: { action: "approve" | "dispute", reason?: string, evidence?: string }
响应: { success, duel_status }
错误: CONFIRM_PERIOD_EXPIRED | NOT_CHALLENGER | ALREADY_CONFIRMED
```

> - approve: 该挑战者确认通过
> - dispute: 提出异议，需附理由和证据
> - 全部挑战者通过 → 立即执行结算（不必等24h）
> - 任一人 dispute → 赌局进入 disputing 状态

#### `POST /api/duel/:id/arbitrate` (管理员)
管理员仲裁争议赌局，三种裁决，不可再申诉。

```
请求: { verdict: "banker_wins" | "banker_loses" | "void", reason: string }
响应: { settlement_details }
```

**裁决处理：**
- `banker_wins`（异议无理）→ 按庄家赢正常结算 + 提出异议的挑战者扣冻结额**10%**罚金（销毁）
- `banker_loses`（庄家虚报）→ 按庄家输正常结算 + 庄家扣押注**10%**罚金（销毁）+ 庄家**禁止坐庄7天**
- `void`（议题模糊，无法判定）→ 赌局作废：
  - 挑战者冻结额**全额退还**（挑战者不罚）
  - 庄家扣押注**10%**罚金（销毁）+ 庄家**禁止坐庄7天**
  - 庄家剩余押注退还
  - 公开赌局入场费不退

#### 自动结算（调度任务）

| 触发条件 | 处理 |
|----------|------|
| 庄家宣布后全部挑战者通过 | 立即按宣布结果结算 |
| 庄家宣布后24h到期，无人异议 | 未操作者视为通过，自动结算 |
| 庄家24h超时未宣布 | 庄家直接判输，按庄家输结算 |
| settle_at到期无挑战者 | 赌局取消，庄家押注退还 |

结算公式（所有正常结算共用）:
- 庄家赢 → 庄家获得：本金退回 + 所有挑战者冻结额（+ 公开赌局已收入场费）
- 庄家输 → 挑战者i获得：冻结额退回 + `庄家押注 × (我的冻结额 / 挑战者冻结总额)`
- 庄家输时仍保留已收的入场费
- 缩头乌龟特效: 输方仍获赢方分成的50%

### 3.9 赛季模块

#### `GET /api/season/current`
当前赛季信息。

```
响应: { season_id, start_date, end_date, days_remaining, status }

赛季排行榜。

```
响应: {
  my_rank, my_percentile,

#### `POST /api/season/settle` (管理员)
赛季结算。余额仅保留 25% 进入下赛季。

```
头像框奖励规则:
  前 3%    -> galaxy_frame (星河头像框)
  3%~10%   -> phantom_frame (幽冥头像框)
  10%~25%  -> aurora_frame (极光头像框)
  25%~40%  -> koi_frame (锦鲤头像框)
```
