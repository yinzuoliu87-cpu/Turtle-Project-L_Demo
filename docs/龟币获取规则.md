# 龟币获取规则（上线版）

两个小游戏的龟币产出与消耗。龟币存储在 `localStorage('petState').coins`，跨游戏共用。

---

## 一、龟龟对战（turtle-battle）

上线只保留三个入口：**在线匹配、房间号对战、深海闯关**（PvE 单人、Boss 模式暂不开放）。

### 1) 在线匹配 / 房间号对战（PvP）

进入对局**收 50 龟币门票**，按"赢家通吃"结算。

| 结果 | 龟币变化 |
|------|---------|
| 胜利 | **+100**（含双方门票，"赢家通吃"） |
| 失败 | **−50**（门票费） |

> 实际净收益：胜 = +50（投入 50 拿回 100）；负 = −50。
> 实现位置：[main.js:1476-1494](games/turtle-battle/js/main.js#L1476-L1494)

### 2) 深海闯关（dungeon，5 关）

按关卡难度发奖，全清后还有结算 bonus。失败则发部分奖励。

#### 关卡奖励（每关通过即结算）
| 关卡 | 龟币 |
|------|------|
| 第 1 关 | 10 |
| 第 2 关 | 20 |
| 第 3 关 | 40 |
| 第 4 关 | 70 |
| 第 5 关（Boss） | 120 |
| **5 关全部基础合计** | **260** |

#### 通关结算 bonus
- **通关 Boss**：+100 龟币
- **零死亡通关**：再 +50 龟币
- **每日首通**：再 +50 龟币

#### 满分一条龙（每日第一次零死亡通关）
`260 + 100 + 50 + 50 = ` **460 龟币**

#### 失败结算
止步于第 N 关时，发放 **已累计奖励 × 50%**（最低 5 龟币保底）。

> 实现位置：关卡发奖 [main.js:2056-2065](games/turtle-battle/js/main.js#L2056-L2065)；最终结算 [main.js:2094-2127](games/turtle-battle/js/main.js#L2094-L2127)

## 二、龟龟跳跳（turtle-jump）

单人闯关：龟从海底向海面上浮，按到达进度结算。

| 结果 | 龟币 |
|------|------|
| 失败（未到顶） | **floor(进度% / 5)** —— 即 5% 进度 = 1 龟币 |
| 成功出海（100%） | **+50** |

> 进度上限 100% 时是通关分支，固定 50。
> 失败分支例：到达 80% → +16 龟币；到达 50% → +10 龟币。
> 实现位置：[game.js:931-932](games/turtle-jump/game.js#L931-L932)、[game.js:964](games/turtle-jump/game.js#L964)

## 三、跨游戏说明

- **存储键**：两边都读写 `localStorage.petState.coins`（同一个键）
- **没有上限 / 衰减**：龟币只增不减（除 PvP 失败扣 50）
- **不同步到服务器**：当前为纯本地存储，清浏览器数据 = 龟币归零
- **充值入口**：暂无

## 四、上线建议（待定）

以下是当前没有但可以考虑的方向，列出来方便讨论：

1. **每日首胜**（PvP）：当前代码里有 `dailyPveWin / dailyBossWin` 的首胜 +50 逻辑（[main.js:1465-1475](games/turtle-battle/js/main.js#L1465-L1475)），但只在 PvE/Boss 触发，PvP 走"赢家通吃"分支 **不享受**。要不要给 PvP 也加每日首胜 bonus？
2. **深海闯关每日次数**：目前可无限刷，是否限每日 1~3 次防刷？
3. **跳跳每日次数 / CD**：同上，否则失败拿 0 龟币几乎没刷子动力，但通关 +50 可以无脑刷。
4. **服务端同步**：上线时是否上账户系统、把 coins 上云？
