# 开战广场（庄家对战系统）需求文档

## 一、产品概述

开战广场是龟投论坛的核心对战模块，采用 **1v多（庄家制）** 模式。用户可以作为庄家发起赌局，或作为挑战者加入他人的赌局，围绕各类事件（体育赛事、财经走势、AI 预测等）进行龟币押注对战。

---

## 二、用户角色

| 角色 | 说明 |
|------|------|
| **庄家** | 发起赌局，定义议题、立场、押注金额、结算时间，负责宣布结果 |
| **挑战者** | 加入赌局，持与庄家相反立场，选择押注金额 |
| **管理员** | 处理争议仲裁（最终裁定，不可上诉） |

---

## 三、核心流程

### 3.1 赌局生命周期

```
创建赌局 → 等待挑战者 → 封盘 → 结算时间到达 → 庄家宣布结果 → 挑战者确认/异议 → 结算完成
```

### 3.2 详细状态

| 状态 | 标识 | 说明 |
|------|------|------|
| 🟢 开放中 | `open` | 接受挑战者加入 |
| 🔒 已封盘 | `sealed` | 不再接受新挑战者（容量满或手动封盘或到达结算时间） |
| ⏳ 待宣布 | `pending` | 结算时间已到，等待庄家24h内宣布结果 |
| ✅ 已结算 | `settled` | 结果确认，龟币已分配 |
| ⚠️ 争议中 | `disputed` | 有挑战者提出异议，等待管理员仲裁 |

## 四、功能模块

### 4.1 创建赌局（庄家）

**表单字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 议题（事件标题） | 文本输入 | ✅ | 赌局主题，需明确可判定 |
| 庄家立场 🔴 | 多行文本 | ✅ | 庄家的预测/观点 |
| 挑战者立场 🔵 | 多行文本 | ✅ | 与庄家相反的立场 |
| 押注金额 | 快捷选择 + 自定义 | ✅ | 100 / 500 / 1,000 / 2,000 / 5,000 / 10,000 🪙 |
| 公开/私人 | 切换按钮 | ✅ | 公开：所有人可见；私人：需邀请码加入 |
| 结算时间 | 日期时间选择器 | ✅ | 到达后自动封盘 |

**业务规则：**
- 庄家押注金额全额冻结
- 创建后不可修改任何字段（议题、立场、金额、时间）
- 仅在零挑战者时可取消（全额退还）
- 有挑战者后不可取消

### 4.2 加入赌局（挑战者）

**加入条件：**
- 赌局状态为"开放中"
- 挑战者冻结总额未超过庄家押注额（容量限制）

**费用结构：**

| 赌局类型 | 入场费 | 说明 |
|----------|--------|------|
| 公开赌局 | 5% | 即时转给庄家（不退还） |
| 私人赌局 | 0% | 无入场费 |

**押注选项：** 50 / 100 / 200 / 500 / 1,000 / 2,000 🪙

**容量机制：**
- 所有挑战者冻结总额 ≤ 庄家押注金额
- 容量满后自动封盘
- 显示容量进度条（已冻结 / 庄家押注额）

### 4.3 封盘机制

触发条件（满足任一）：
1. 挑战者冻结总额达到庄家押注额（自动封盘）
2. 庄家手动封盘（仅私人赌局）
3. 到达结算时间（自动封盘）

### 4.4 结算流程

#### 第一步：庄家宣布结果
- 结算时间到达后，庄家需在 **24小时内** 宣布结果
- 选项：庄家赢 / 庄家输
- **超时未宣布 = 自动判定庄家输**

#### 第二步：挑战者确认
- 每位挑战者有 **24小时** 确认或提出异议
- 沉默（超时未操作）= 默认确认

#### 第三步：结算 / 仲裁
- 全部确认 → 立即结算
- 任一挑战者提出异议 → 进入管理员仲裁（24h内裁定，最终决定不可上诉）

### 4.5 赢输计算

**庄家赢：**
```
庄家收益 = 退还本金 + 所有挑战者冻结金额 + 已收入场费
```

**庄家输：**
```
每位挑战者收益 = 退还本金 + 庄家押注额 × (我的冻结额 / 所有挑战者冻结总额)
```

### 4.6 异议系统

**提交异议（挑战者）：**

底部弹出面板，包含：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 异议类型 | 单选 | ✅ | 🚫 虚假结果 / ❓ 议题模糊 / 📝 其他 |
| 详细描述 | 多行文本 | ✅ | 异议原因说明 |
| 证据链接 | URL 输入 | 否 | 支持性证据链接 |
| 图片证据 | 文件上传 | 否 | 最多3张图片，支持拖拽 |

**虚假异议惩罚：** 扣除异议者冻结金额的 10%

---

## 五、违规处罚

| 违规行为 | 仲裁结果 | 处罚 |
|----------|----------|------|
| 庄家虚假宣布结果 | banker_loses | 正常输钱赔付 + 扣除本金 10% + 封号7天 |
| 议题模糊不可判定 | void（作废） | 挑战者全额退还，庄家扣除 10% 本金 + 封号7天（入场费不退） |
| 挑战者虚假异议 | banker_wins | 正常庄家赢赔付 + 扣除异议者冻结额 10% |

## 七、庄家提示（我做的庄 Tab）

1. 📝 **议题要清晰** — 避免歧义，用"≥3500点"不用"涨很多"，模糊议题可能被仲裁判无效（扣10%+封号7天）
2. ⏰ **24小时宣布期限** — 结算时间到后必须24h内宣布结果，超时自动判输，建议设提醒
3. 🔒 **有人加入不可取消** — 一旦有挑战者加入，赌局锁定不可撤销，创建前请三思
4. 💰 **入场费归你** — 公开赌局挑战者入场费(5%)即时到账，即使你输了也不退
5. ✅ **诚实宣布** — 虚假宣布结果将扣除10%本金 + 封号7天

## 八、挑战者提示（我的挑战 Tab）

1. ⏰ **关注结算时间** — 庄家宣布结果后你有24h确认，沉默=默认确认
2. ⚠️ **异议需有据** — 虚假异议将被扣除冻结金额的10%

## 十、数据模型（建议）

### Duel（赌局）
```
{
  id: string,
  bankerId: string,          // 庄家用户ID
  title: string,             // 议题
  bankerStance: string,      // 庄家立场
  challengerStance: string,  // 挑战者立场
  bankerStake: number,       // 庄家押注额
  visibility: 'public' | 'private',
  inviteCode: string | null, // 私人赌局邀请码
  settlementTime: datetime,  // 结算时间
  status: 'open' | 'sealed' | 'pending' | 'settled' | 'disputed',
  declaredResult: 'win' | 'lose' | null,
  finalResult: 'banker_wins' | 'banker_loses' | 'void' | null,
  totalFrozen: number,       // 挑战者冻结总额
  createdAt: datetime,
  sealedAt: datetime | null,
  declaredAt: datetime | null,
  settledAt: datetime | null
}
```

### Challenge（挑战记录）
```
{
  id: string,
  duelId: string,
  challengerId: string,
  amount: number,            // 押注额
  entryFee: number,          // 入场费
  frozen: number,            // 冻结额（= amount）
  confirmed: boolean | null, // true=确认, false=异议, null=待确认
  createdAt: datetime
}
```

### Dispute（异议）
```
{
  id: string,
  duelId: string,
  challengerId: string,
  type: 'fake_result' | 'vague_topic' | 'other',
  description: string,
  evidenceUrl: string | null,
  images: string[],          // 图片URL列表（最多3张）
  adminVerdict: 'banker_wins' | 'banker_loses' | 'void' | null,
  createdAt: datetime,
  resolvedAt: datetime | null
}
```
