# 🐢 龟投 · 宠物 后端需求文档

**v3.3 · 2026年3月**

> 本文档面向后端开发工程师，描述龟投 App 宠物系统所需的数据结构、业务逻辑、接口规范及待确认事项。前端/产品/后端应在开发前共同对齐"待确认问题"一节中的所有条目。

---

## 1. 文档说明

### 1.1 功能模块一览

- **宠物展示**：龟种外观 × 场景 × 披风，展示在宠物页面上方
- **底部导航（4页）**：能力 / 龟种 / 装扮 / 记忆
- **体力系统**：上限 100，AI 对话消耗，龟粮恢复
- **AI 对话**：接入千问大模型，消耗体力，记录历史
- **开蛋系统**：500 龟币/次，六档稀有度（C/B/A/S/SS/SSS），无保底机制
- **成就系统**：披风与场景通过完成特定成就解锁
- **火花系统**：连续登录累计火花加成每日登录奖励，龟种切换与火花无关，每天仅可换一只龟
- **小游戏联动**：龟种能力同步至小游戏（护盾 / 体积缩小）
- **好友系统**：主页内展示好友列表，支持好友私人对赌
- **赛季系统**：每 3 个月一赛季，按龟币排名发放稀有龟种与限定披风

### 1.2 好友私人对赌规则

- 发起方设定话题与下注金额，对方可自由设定己方下注金额（双方金额可不同）；结果以话题结算为准
- **结算示例**：A 下注 100，B 下注 70，A 获胜 → A 赢得 B 的 70 龟币
- 若 A 装备海盗龟/赛博龟，额外从 B 账户窃取其输掉金额的 10%/15%，若 B 余额不足则取剩余全部
- **窃取示例（赛博龟）**：B 输了 70，额外窃取 70 × 15% = 10 龟币（向下取整）；若 B 余额为 0 则窃取 0

### 1.3 龟币消耗概览

- 主要消耗：投票事件下注（用户自由决定金额）
- 次要消耗：开蛋（500 龟币/次）、购买龟粮（20 龟币/个）

## 3. 体力系统

### 3.1 基本规则

- 体力上限：100
- AI 对话消耗：每次 -5（装备无头龟时 -4）
- 体力不足时，拒绝对话请求并返回提示

### 3.2 体力回复

- 每小时回复 5 点，上限 100
- 体力满时停止计算，不溢出

> **后端实现建议**：存储 `stamina_current` 和 `stamina_last_regen_at`，查询时实时计算当前值（每小时+5），无需定时任务。

### 3.3 龟粮（苹果）

- 一个苹果售价：20 龟币
- 一个苹果回复体力：25 点（不超上限）
- 糖果龟装备时购买折扣：-20%，即 **16 龟币/个**

## 4. 开蛋系统

### 4.1 抽取规则

- 单次费用：500 龟币（忍者龟 475 / 泡泡龟 450 / 水晶龟 410）
- 可出任意龟种，包括已拥有的
- 无保底机制

**稀有度概率分布：**

| 稀有度 | 概率 | 龟种（数量） |
|--------|------|-------------|
| C | 40% | 小龟 / 石头龟 / 竹叶龟（3只，均分） |
| B | 30% | 天使龟 / 寒冰龟 / 忍者龟 / 双头龟 / 幽灵龟 / 钻石龟 / 财神龟 / 骰子龟（8只，均分） |
| A | 15% | 彩虹龟 / 赌神龟 / 猎人龟 / 海盗龟 / 糖果龟 / 泡泡龟 / 线条龟 / 闪电龟（8只，均分） |
| S | 10% | 凤凰龟 / 熔岩龟 / 赛博龟 / 水晶龟 / 宝箱龟 / 星际龟（6只，均分） |
| SS | 4% | 缩头乌龟 / 无头龟（2只，均分） |
| SSS | 1% | 龟壳（1只） |

### 4.2 重复龟种返还规则

- 若开蛋结果为已拥有龟种，自动返还本次费用的 **30%**
- 普通返还：500 × 30% = **150 龟币**
- 忍者龟折扣：475 × 30% = **142 龟币**（向下取整）
- 泡泡龟折扣：450 × 30% = **135 龟币**
- 水晶龟折扣：410 × 30% = **123 龟币**（向下取整）
- 响应体需包含 `is_duplicate: Boolean` 和 `refund_amount: Integer` 字段

## 5. 龟壳（SSS）及特殊金融能力

龟壳（SSS）与闪电龟（A）均提供欠账能力；存款生息为龟壳与星际龟（S）专属。

### 5.1 欠账系统

| 龟种 | 欠账上限 | 每日补贴 |
|------|----------|----------|
| 龟壳（SSS） | -1000 龟币 | 欠款额 × 22% |
| 闪电龟（A） | -300 龟币 | 欠款额 × 25% |

- 余额可正常显示负数，如「-320 🪙」
- 无利息，欠款期间不产生额外费用
- **锁龟机制**：余额 < 0 时禁止切换龟种，返回错误码 `DEBT_UNPAID`

**补贴示例：**
- 龟壳：余额 -500 → 补贴 110 龟币，余额变为 -390
- 闪电龟：余额 -200 → 补贴 50 龟币，余额变为 -150

### 5.2 存款生息系统

| 龟种 | 利率 | 每日上限 |
|------|------|----------|
| 龟壳（SSS） | 余额 × 5% | 1000 龟币 |
| 星际龟（S） | 余额 × 3% | 1000 龟币 |

- 生息与补贴互斥：余额 < 0 触发补贴，余额 > 0 触发生息，余额 = 0 两者均不触发

### 5.3 结算时机

- 装备龟壳/闪电龟后，从**次日北京时间 0 点**起开始触发补贴
- 装备星际龟/龟壳后，从**次日 0 点**起开始触发生息
- 切换龟种后当日不再结算原龟种的补贴/生息，新龟种从次日 0 点生效
- 与每日登录奖励同批次结算

### 5.4 新增接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `POST /pet/loan` | POST | 借款：余额扣减至最低限额，超限返回 `LOAN_LIMIT_EXCEEDED` |
| `POST /daily/shell-settle` | POST | 每日结算：发放补贴/生息，每日仅触发一次 |

- 两个计数相互独立，达到条件立即解锁披风，计数不重置

## 8. 火花系统

### 8.1 触发条件

- 用户连续两天及以上完成每日首次登录即可累积火花
- 与当前装备龟种无关，切换龟种不影响火花计数

### 8.2 计数规则

- 每满足一天连续登录，`login_streak` +1
- 中断连续登录（跳过一天），`login_streak` 归零

### 8.3 奖励计算

$$当日登录总奖励 = 100（基础）+ 5 × login\_streak（火花加成）$$

**例**：`login_streak = 3` → 当日奖励 = 100 + 15 = **115 龟币**

### 8.4 龟种每日切换限制

- 每位用户每天（北京时间）仅可切换龟种一次
- 后端以 `last_equip_date` 记录上次切换日期，切换前校验
- 当日已切换返回错误码 `EQUIP_DAILY_LIMIT`；披风和场景切换不受此限制

> ⚠️ **待确认**：每日判断时区以北京时间（UTC+8）0 点为准，请后端确认并统一。

## 9. AI 对话

### 9.1 调用流程

- 前端发起对话 → 后端扣除体力（stamina consume 接口）→ 后端调用千问 API → 返回结果并存储
- 体力不足（< 5，无头龟 < 4）时，返回错误码，不调用 AI

### 9.2 对话存储

- 每条记录包含：角色（user/assistant）、内容、时间戳
- 存储在 `ai_chat_history` 字段（或独立表）
- 供「记忆」页面调用展示

> ⚠️ **待确认**：AI 调用由后端统一对接千问 API，还是前端直接调用？建议后端调用以保护 API Key。

> ⚠️ **待确认**：记忆页面具体展示内容：全量对话记录？每日摘要？关键词标签？请产品确认。

## 10. 每日登录 & 签到

- 触发时机：用户每日北京时间首次打开 App，调用 `POST /auth/daily-login`
- 后端检查 `last_login_date`，若非当日则视为新的一天登录
- 奖励展示：应用内签到界面显示「100 + 火花加成」
- 同时计算火花逻辑，返回本次奖励龟币数量（基础 100 + 5 × login_streak）

## 11. 小游戏联动（下海捕捞）

### 11.1 每日龟币奖励

- 每位用户每天**第一局**结束后，发放龟币 = 本局分数 ÷ 400（向下取整）
- 例：得分 2,000 → 奖励 5 龟币；得分 399 → 奖励 0 龟币
- 仅第一局触发，当日后续游戏不再发放
- 宝箱龟装备时，小游戏龟币奖励额外 +25%

### 11.2 龟种能力同步规则

| 级别 | 小游戏能力 |
|------|-----------|
| C / B / A | 仅下海捕捞，无额外加成 |
| S | 下海捕捞 + 护盾 ×1（每局免疫一次碰撞） |
| SS / SSS | 下海捕捞 + 体积缩小 30% |

### 11.3 接口建议

- `POST /game/finish`：接收本局得分，判断是否首局，计算并发放龟币
- `GET /user/game-ability`：返回当前龟种的游戏能力标识（`shield` / `shrink` / `none`）
- 能力判断建议在服务端完成，防止客户端篡改

## 13. 赛季系统

每三个月为一个赛季，赛季结束时按用户龟币排名发放稀有龟种与限定披风。

### 13.1 赛季周期

- 每赛季时长：3 个月（约 90 天）
- 结算时机：赛季结束日北京时间 0 点，与每日登录奖励同批次处理
- 参与条件：赛季内至少完成 1 次投票下注（防止僵尸账号占位）

### 13.2 排名奖励

| 排名区间 | 龟种奖励 | 限定披风 | 说明 |
|----------|----------|----------|------|
| 前 3% | SSS 随机 | ⭐ 星河披风 | 已拥有该龟种则返还 500 龟币 |
| 3% ~ 10% | SS 随机 | 👻 幽冥披风 | 已拥有该龟种则返还 350 龟币 |
| 10% ~ 25% | S 随机 | 🌌 极光披风 | 已拥有该龟种则返还 200 龟币 |
| 25% ~ 40% | A 或 B 随机 | 🐟 锦鲤披风 | 已拥有该龟种则返还 100 龟币 |
| 40% 以下 | — | — | 无奖励，可获参与徽章（待定） |

### 13.3 限定披风说明

- 限定披风仅通过赛季奖励获得，不可通过成就解锁或开蛋获得
- 已拥有限定披风时，重复发放不返还龟币

### 13.4 排名计算

- 排名依据：赛季结束时用户的龟币余额（含负数，负数排名靠后）
- 同分处理：赛季内更早达到该余额的用户排名更高
- 排名百分比 = 用户排名 ÷ 本赛季参与总人数
- 钻石龟（B 级）可使排名百分比额外 +2%

### 13.5 赛季结束余额保留规则

- 赛季结束结算后，龟币余额**保留 25%**，其余 75% 清零
- 例：赛季末 10,000 龟币 → 新赛季初始余额 **2,500 龟币**
- 负数同规则：-400 → 新赛季初始余额 **-100 龟币**
- 保留后余额写入 `turtle_coins`，`season_start_balance` 记录该值
