## 一、页面概述

**页面路径：** `pet.html`
**入口：** 主站左侧栏宠物展示区点击跳转

## 三、Tab 0：能力

### 3.1 体力系统

- 体力上限：100
- AI 对话消耗：每次 -5（装备无头龟时 -4）
- 自然回复：每小时 +5，上限 100
- 龟粮（苹果）：20 龟币/个，回复 25 体力（糖果龟装备时 16 龟币/个）

### 3.2 当前龟种能力展示

- 小游戏能力：none / shield / shrink
- 特殊能力：每只龟种独有的经济/战斗加成

### 3.3 开蛋系统

- 单次费用：500 龟币（忍者龟 475 / 泡泡龟 450 / 水晶龟 410）
- **可出任意龟种，包括已拥有的**
- 重复龟种返还本次费用的 **30%**
- 无保底机制

**稀有度概率分布（6档）：**

| 稀有度 | 概率 | 龟种数量 |
|--------|------|---------|
| SSS    | 1%   | 1       |
| SS     | 4%   | 2       |
| S      | 10%  | 6       |
| A      | 15%  | 8       |
| B      | 30%  | 8       |
| C      | 40%  | 3       |

## 六、Tab 3：记忆

- 对话界面（气泡式），每次对话消耗体力

## 七、火花系统

- 连续登录天数累计 `loginStreak`
- 中断归零
- 每日登录奖励 = 100（基础）+ 5 x loginStreak
- 与龟种切换无关

## 八、数据持久化

### 8.1 localStorage Key

| Key | 写入方 | 读取方 | 内容 |
|-----|--------|--------|------|
| `petState` | pet.html | game.html | 龟种/披风/场景/龟币/体力/火花/聊天记录 |
| `turtleVoteStreak` | script.js, battle.js | pet.html | 有效投票连胜（>=100龟币） |
| `turtleUpsetStreak` | script.js, battle.js | pet.html | 爆冷连胜次数 |
| `turtleGameBest` | game.html | pet.html | 小游戏历史最高分 |
| `turtleTotalVotes` | script.js, battle.js | pet.html | 累计投票次数 |

## 九、小游戏联动（game.html）

### 9.1 龟种能力同步（3档）

| 级别 | 小游戏能力 |
|------|-----------|
| C / B / A | 仅下海捕捞，无额外加成 |
| S | 下海捕捞 + 护盾x1（每局免疫一次碰撞） |
| SS / SSS | 下海捕捞 + 体积缩小 30% |

### 9.2 分数回写

- 游戏结束时将最高分写入 `turtleGameBest`
- 用于披风解锁（暗黑 15000+ / 樱花 20000+）

## 十、投票连胜联动

- 主站投票后调用 `recordVoteResult(betAmount)`
- `turtleTotalVotes` +1
- 若 `betAmount >= 100`：
  - 70% 概率判定为胜 -> `turtleVoteStreak` +1
  - 猜错 -> `turtleVoteStreak` 归零
  - 若事件胜率 < 25% 且猜对 -> `turtleUpsetStreak` +1
  - 猜错 -> `turtleUpsetStreak` 归零
