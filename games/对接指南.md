# 龟龟游戏模块 — 主站对接指南

## 模块概述

两个独立的小游戏模块，可嵌入任何主站：

| 模块 | 目录 | 说明 |
|------|------|------|
| 龟龟对战 | `games/turtle-battle/` | 2V2回合制策略战斗 |
| 龟龟出海 | `games/turtle-jump/` | 跳跃类休闲小游戏 |

## 对接方式

### 方式1：iframe嵌入（最简单）

```html
<!-- 龟龟对战 -->
<iframe src="/games/turtle-battle/index.html" 
        style="width:100%;height:100vh;border:none"></iframe>

<!-- 龟龟出海 -->
<iframe src="/games/turtle-jump/index.html"
        style="width:100%;height:100vh;border:none"></iframe>
```

优点：零改动，完全隔离  
缺点：无法共享主站导航/用户系统

### 方式2：直接部署为子路径（推荐）

将文件放到主站的子目录：
```
主站根目录/
├── games/
│   ├── turtle-battle/    ← 整个目录复制
│   ├── turtle-jump/      ← 整个目录复制
│   └── 对接指南.md
├── assets/
│   └── pets/             ← 共享龟龟图片
└── index.html            ← 主站首页
```

需要修改的地方：

#### 1. 返回链接
`turtle-battle/index.html` 第13行：
```html
<a class="back-link" href="../../index.html">← 返回龟投</a>
```
改为主站首页的实际路径。

#### 2. 龟龟图片路径
`turtle-battle/js/pets.js` 中所有龟的图片路径使用相对路径 `../../assets/pets/xxx.png`。  
如果目录结构不同，需要全局替换。

`turtle-jump/game.js` 同理，引用路径为 `../../assets/pets/xxx.png`。

#### 3. 龟币系统连接
两个模块通过 `localStorage` 的 `petState` 键共享龟币：
```javascript
// 读取龟币
const ps = JSON.parse(localStorage.getItem('petState') || '{}');
const coins = ps.coins || 0;

// 写入龟币
ps.coins = newAmount;
localStorage.setItem('petState', JSON.stringify(ps));
```

如果主站有自己的用户系统，需要将 `localStorage` 替换为主站的API调用。

## 依赖项

### 外部CDN（仅龟龟对战）
```html
<script src="https://cdn.jsdelivr.net/npm/peerjs@1/dist/peerjs.min.js" defer></script>
```
PeerJS 用于线上P2P对战。如果不需要线上对战功能，可以删除。

### TURN服务器（线上对战用）
`main.js` 中配置了TURN服务器用于NAT穿透：
```javascript
const TURN_SERVERS = [...]; // 需要配置有效的TURN服务器
```
如果不使用线上对战，可忽略。

### 无其他外部依赖
- 无npm包
- 无构建工具
- 纯原生JS/CSS/HTML

## localStorage 键名

| 键名 | 模块 | 说明 |
|------|------|------|
| `petState` | 共享 | 龟币、拥有的龟等 `{coins:100, owned:{...}}` |
| `turtleBattleRecord` | 对战 | 战绩 `{wins:10, losses:5}` |
| `tutorialComplete` | 对战 | 新手引导完成标记 |
| `dailyPveWin` | 对战 | 每日PVE首胜日期 |
| `dailyBossWin` | 对战 | 每日Boss首胜日期 |
| `turtleBestProg` | 出海 | 最高进度 |
| `turtleAttempts` | 出海 | 挑战次数 |

如果主站使用后端用户系统，建议将这些数据迁移到服务端存储。
