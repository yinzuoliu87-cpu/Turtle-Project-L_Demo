# 龟龟对战 — 后端需求文档

## 项目概述

龟龟对战是一个2V2回合制战斗网页游戏，当前为纯前端实现。需要后端支持以下功能：
- 全服排行榜（胜率、最强组合）
- 用户账号/战绩持久化
- 未来可能的匹配对战服务

---

## 一、用户系统

### 1.1 用户标识
- 目前无登录系统，建议支持以下方式：
  - **游客模式**：自动生成UUID，localStorage存储
  - **微信登录**（可选）：OAuth2.0
  - **用户名+密码**（可选）

### 1.2 用户数据结构
```json
{
  "userId": "uuid-xxx",
  "nickname": "龟大师",
  "avatar": "default",

```

## 二、排行榜系统

### 2.1 全服胜率排行

**API: GET /api/leaderboard/winrate**

请求参数：
| 参数 | 类型 | 说明 |
|------|------|------|
| limit | number | 返回条数（默认50） |
| offset | number | 分页偏移 |
| minGames | number | 最低场次过滤（默认10） |

响应：
```json
{
  "leaderboard": [
    {
      "rank": 1,
      "userId": "xxx",
      "nickname": "龟大师",
      "wins": 85,
      "totalGames": 150,
      "winRate": 0.567,
      "favoriteTeam": ["rainbow", "lightning"]
    }
  ],
  "myRank": 42,
  "total": 1000
}
```

### 2.2 最强组合排行

**API: GET /api/leaderboard/combos**

统计维度：2只龟的组合胜率（不分先后顺序）

响应：
```json
{
  "combos": [
    {
      "rank": 1,
      "team": ["rainbow", "lightning"],
      "teamNames": ["彩虹龟", "闪电龟"],
      "wins": 320,
      "totalGames": 500,
      "winRate": 0.640,
      "players": 89
    }
  ]
}
```

### 2.3 单龟使用率/胜率排行

**API: GET /api/leaderboard/turtles**

响应：
```json
{
  "turtles": [
    {
      "id": "rainbow",
      "name": "彩虹龟",
      "rarity": "A",
      "pickRate": 0.15,
      "winRate": 0.54,
      "totalGames": 5000
    }
  ]
}
```

## 三、战斗记录

### 3.1 上报战斗结果

**API: POST /api/battle/report**

请求体：
```json
{
  "userId": "xxx",
  "mode": "pve",
  "result": "win",
  "duration": 12,
  "team": ["rainbow", "lightning"],
  "enemyTeam": ["lava", "crystal"],
  "teamDmg": [450, 380],
  "enemyDmg": [320, 290],
  "timestamp": "2026-04-05T12:00:00Z"
}
```

### 3.2 查询个人战绩

**API: GET /api/battle/history?userId=xxx&limit=20**

响应：
```json
{
  "battles": [
    {
      "id": "battle-xxx",
      "mode": "pve",
      "result": "win",
      "team": ["rainbow", "lightning"],
      "enemyTeam": ["lava", "crystal"],
      "duration": 12,
      "timestamp": "2026-04-05T12:00:00Z"
    }
  ],
  "total": 150
}
```

## 四、数据统计（管理后台用）

### 4.1 全局统计
- 每日活跃用户数（DAU）
- 每日对局数
- 各龟使用率/胜率分布
- 平均对局时长
- 新用户留存率

### 4.2 建议的统计表

**battles 表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 用户ID |
| mode | ENUM | pve/pvp/boss |
| result | ENUM | win/loss |
| team | JSON | ["rainbow","lightning"] |
| enemy_team | JSON | ["lava","crystal"] |
| team_dmg | JSON | [450, 380] |
| enemy_dmg | JSON | [320, 290] |
| duration | INT | 回合数 |
| created_at | TIMESTAMP | 创建时间 |

**users 表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| nickname | VARCHAR | 昵称 |
| total_games | INT | 总场次 |
| wins | INT | 胜场 |
| created_at | TIMESTAMP | 注册时间 |
| last_active | TIMESTAMP | 最后活跃 |

**combo_stats 物化视图/缓存**
- 按 team 组合（排序后）聚合 wins/total
- 每小时或每100场刷新

## 五、技术建议

### 推荐技术栈
- **API**: Node.js + Express / Python + FastAPI
- **数据库**: PostgreSQL（排行榜查询性能好）
- **缓存**: Redis（排行榜热数据缓存）
- **部署**: Docker + 云服务器 / Serverless

### 性能要求
- 排行榜查询 < 200ms
- 战斗上报 < 100ms
- 支持 1000 并发用户

### 安全考虑
- 战斗结果需要服务端校验（防作弊）
- API 限流（每用户每秒最多5次）
- 用户数据加密存储

## 六、前端对接方式

前端会在以下时机调用后端API：
1. **页面加载时**：查询排行榜、获取个人战绩
2. **战斗结束时**：上报战斗结果
3. **排行榜页面**：分页查询排行数据

前端会提供统一的 API 调用层：
```javascript
// config
const API_BASE = 'https://api.example.com';

// 上报战斗
async function reportBattle(data) { ... }

// 获取排行榜
async function getLeaderboard(type, params) { ... }

// 获取个人战绩
async function getMyStats() { ... }
```

- C级(3只)：小龟、石头龟、竹叶龟

- SS级(2只)：缩头乌龟、无头龟
