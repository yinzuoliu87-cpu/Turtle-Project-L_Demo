# 移动端适配方案

## 一、核心问题与当前实现状态

### 1.1 浏览器占用空间 -- 已解决

**问题**：Chrome/Safari的顶部地址栏 + 底部导航栏大约占屏幕高度的 15-20%。

**已实现方案**：
- 使用 `100dvh`（dynamic viewport height）而非 `100vh`
- `body` 设置 `min-height:100dvh; overflow-x:hidden`
- 移动端战斗界面 `height:100dvh; display:flex; flex-direction:column`

---

### 1.2 防止误触刷新/页面乱动 -- 已解决

**已实现方案**：

CSS层面：
```css
html { overscroll-behavior: none; -webkit-overflow-scrolling: auto; }
body { overscroll-behavior: none; }
```

JS层面：
```js
// 战斗界面：禁止默认触摸行为（index.html中）
document.addEventListener('touchmove', function(e) {
  const battle = document.getElementById('screenBattle');
  if (battle && battle.classList.contains('active')) {
    if (!e.target.closest('.scrollable, .action-panel, ...')) e.preventDefault();
  }
}, { passive: false });
```

各滚动容器已添加 `overscroll-behavior: contain`：
- `.fighter-detail-panel`（底部详情面板）
- `.action-panel`（技能/选龟面板）
- 战斗日志面板

**分场景处理**：
| 场景 | 滚动策略 | 状态 |
|------|---------|------|
| 主菜单 | 锁定，不需要滚动 | 已实现 |
| 选龟界面 | 龟列表区域可滚动，阵型格子固定 | 已实现 |
| 战斗界面 | 完全锁定，所有操作通过点击 | 已实现 |
| 战斗日志 | 弹出面板内部可滚动 | 已实现 |
| 结果界面 | 锁定 | 已实现 |

---

### 1.3 战斗背景方案 -- 已采用方案A

采用 **方案A：一套背景，CSS适配**：
- 背景图保持 16:9 原图
- `.battle-scene` 使用 `background-size:cover; background-position:center` 裁切
- PC端保持 `aspect-ratio:16/9; max-height:70vh`
- 移动端 `aspect-ratio:auto; flex:1; min-height:0; max-height:none`

---

## 二、当前战斗布局实现

### 2.1 场景架构 (scene-based)

当前已完成从"卡片式"到"场景式"的重构：

```
.battle-scene（背景容器，cover-mapped定位）
├── .scene-labels（顶部标签：左方/右方）
├── .scene-turtle（每只龟，absolute定位在背景上）
│   ├── HP条
│   ├── 精灵图
│   ├── 名字
│   └── 状态图标
└── ::before（边缘渐变遮罩）
```

**关键CSS**：
```css
.battle-scene {
  position: relative; width: 100%;
  aspect-ratio: 16/9; max-height: 70vh;
  border-radius: 12px; overflow: hidden;
  background-color: #0a1628;
  background-size: cover; background-position: center;
}
/* scene-inner 已移除 — 龟直接通过 cover mapping 定位 */
```

### 2.2 龟的定位（cover-mapped）

龟不再使用DOM流布局，而是通过 `position:absolute` + 百分比 `left/bottom` 定位到战场上。每只龟的位置根据阵型格子（front-0/1/2, back-0/1/2）映射到场景百分比坐标。

移动端通过 `transform:scale()` 缩放：
```css
@media (max-width: 768px) {
  .scene-turtle { transform: scale(.5) }
  .scene-turtle:hover { transform: scale(.55) }
}
@media (max-width: 480px) {
  .scene-turtle { transform: scale(.5) }
  .scene-turtle:hover { transform: scale(.55) }
}
```

### 2.3 技能面板（action-panel）

移动端技能面板为底部弹出式：
```css
@media (max-width: 768px) {
  .action-panel { overscroll-behavior: contain; display: none; }
  .action-panel.show { display: block; overscroll-behavior: contain; }
}
```

### 2.4 详情面板（bottom sheet）

龟的详情面板在移动端变为底部上滑式：
```css
@media (max-width: 768px) {
  .fighter-detail-panel {
    position: fixed !important;
    bottom: 0 !important;
    width: 100% !important;
    max-height: 60vh;
    border-radius: 16px 16px 0 0;
    animation: slideUpSheet .2s ease;
    overscroll-behavior: contain;
    -webkit-overflow-scrolling: touch;
  }
}
```

## 四、PWA 化（可选，未实施）

如果要做到"像APP一样"完全不显示浏览器UI：

```json
// manifest.json
{
  "display": "standalone",
  "orientation": "portrait"
}
```

```html
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
```

这样用户"添加到主屏幕"后，打开就是全屏应用，没有地址栏和底部栏。
