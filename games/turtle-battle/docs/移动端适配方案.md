# 移动端适配方案

## 一、核心问题分析

### 1.1 浏览器占用空间

**Chrome/Safari的顶部地址栏 + 底部导航栏**大约占屏幕高度的 15-20%：
- Chrome Android：地址栏 ~56px + 底部工具栏 ~48px = ~104px
- Safari iOS：地址栏 ~50px + 底部 Home 条 ~34px = ~84px
- 实际可用高度 ≈ 80-85% 屏幕高度

**解决方案**：
- 使用 `100dvh`（dynamic viewport height）而不是 `100vh`
- `dvh` 会自动适应浏览器 UI 的显示/隐藏
- 战斗界面设为 `height: 100dvh; overflow: hidden` 防止整页滚动

---

### 1.2 防止误触刷新/页面乱动

**问题**：用户上下滑动时可能触发：
- Chrome 下拉刷新（pull-to-refresh）
- Safari 橡皮筋弹性滚动（rubber band）
- 页面整体滚动导致布局错位

**参考**：Bing 首页、微信小程序、PWA 应用

**解决方案**：

```css
/* 全局防止 overscroll */
html, body {
  overscroll-behavior: none;        /* 禁止下拉刷新 */
  -webkit-overflow-scrolling: auto; /* 禁止 iOS 橡皮筋 */
  position: fixed;                  /* 战斗时锁定 body */
  width: 100%;
  height: 100dvh;
  overflow: hidden;
}

/* 只在需要滚动的容器内允许滚动 */
.scrollable {
  overflow-y: auto;
  -webkit-overflow-scrolling: touch;
  overscroll-behavior: contain;     /* 滚动不传播到父级 */
}
```

```js
// 战斗界面：禁止默认触摸行为
document.addEventListener('touchmove', (e) => {
  if (document.getElementById('screenBattle')?.classList.contains('active')) {
    // 只允许在可滚动区域内滚动
    if (!e.target.closest('.scrollable')) e.preventDefault();
  }
}, { passive: false });
```

**分场景处理**：
| 场景 | 滚动策略 |
|------|---------|
| 主菜单 | 锁定，不需要滚动 |
| 选龟界面 | 龟列表区域可滚动，阵型格子固定 |
| 战斗界面 | 完全锁定，所有操作通过点击 |
| 战斗日志 | 全屏弹出，内部可滚动 |
| 结果界面 | 锁定 |

### 1.3 战斗背景方案

**方案A：一套背景，CSS适配**（推荐）
- 背景图保持 16:9 原图
- 移动端通过 `background-size: cover; background-position: center` 裁切两侧
- 龟的位置用百分比，自动适应
- 优点：不需要额外美术资源，维护简单
- 缺点：横向两侧被裁切，但战斗区域集中在中间所以影响不大

**方案B：两套背景**
- 桌面端 16:9，移动端 9:16 竖版
- 移动端布局改为上下对阵（我方在下，敌方在上）
- 需要重新设计龟的站位
- 优点：视觉效果最佳
- 缺点：工作量大，需要新的美术资源

**方案C：响应式裁切 + 补充元素**
- 背景图用宽幅（21:9），桌面和移动端都能裁切使用
- 移动端裁切为接近 3:2 的区域
- 用CSS渐变/遮罩补充边缘
- 优点：一张图适配所有尺寸
- 缺点：需要重新制作背景图

**建议选方案A**，当前背景已经能cover适配。

### 2.2 空间分配

| 区域 | 高度占比 | 说明 |
|------|---------|------|
| 回合信息栏 | 5% | 回合数 + 计时器 + 功能按钮 |
| 战斗场景 | 45-50% | 背景 + 6只龟 + 血条 |
| 技能/选龟面板 | 35-40% | 底部弹出，可滚动 |
| 行动提示 | 5% | 当前行动龟 + 换龟按钮 |

### 2.3 龟的占位（移动端）

当前方案：用CSS `transform: scale(0.45)` 缩放PC布局。

**优化方向**：
- 移动端场景比例改为 **3:2**（而不是16:9），给龟更多纵向空间
- 前排龟靠中间，后排龟靠两侧
- 龟的名字在移动端可以隐藏，只显示血条
- 点击龟弹出底部详情面板（已实现）

## 三、技能面板设计

### 3.1 参考游戏

| 游戏 | 技能面板位置 | 特点 |
|------|------------|------|
| 怪物圣所 | 底部固定 | 横排3个技能按钮，点击释放 |
| 梦幻西游手游 | 底部弹出 | 竖排技能列表，带简述 |
| 阴阳师 | 底部固定 | 圆形技能图标，长按查看详情 |
| 云顶之弈 | 不需要（自动战斗） | - |
| 放置少女 | 底部固定 | 大按钮，一键释放 |

### 3.2 推荐方案

**底部固定面板**（当前实现接近）：
- 技能卡片竖排排列
- 每个卡片显示：技能名 + brief描述
- 点击整个卡片释放
- CD中的技能灰色+倒计时
- 连携技金色卡片在最下方

**优化点**：
- 技能面板最高不超过 40vh
- 超出可滚动（overscroll-behavior: contain）
- 选龟时底部弹出选龟面板，选完自动切换为技能面板
- 选目标时技能面板隐藏，场景全屏显示发光龟

## 四、触摸交互优化

### 4.1 手势映射

| 手势 | 功能 |
|------|------|
| 点击龟 | 显示详情面板 / 选择目标 |
| 点击技能卡 | 释放技能 |
| 点击格子 | 选中格子（放龟） |
| 点击龟卡 | 放入选中格子 / 自动放入 |
| 左右滑动 | 无功能（防止误触） |
| 上下滑动 | 在可滚动区域内滚动 |
| 长按 | 无功能（避免与系统菜单冲突） |

### 4.2 触摸区域最小尺寸

- 所有可点击元素最小 44×44px（iOS HIG标准）
- 龟的点击区域通过padding扩展（已实现）
- 技能按钮最小高度 48px

## 五、PWA 化（可选）

如果要做到"像APP一样"完全不显示浏览器UI：

```json
// manifest.json
{
  "display": "standalone",
  "orientation": "portrait"
}
```

```html
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
```

这样用户"添加到主屏幕"后，打开就是全屏应用，没有地址栏和底部栏。
