# 龟龟对战 — 模块总结文档

## 项目概述

龟龟对战是一个 2V2 回合制策略战斗网页游戏，玩家选择2只龟组成队伍，与AI或其他玩家对战。

**技术栈**：纯前端 HTML/CSS/JS，PeerJS 实现 P2P 线上对战  
**文件结构**：
```
games/turtle-battle/
├── index.html          # 主页面
├── style.css           # 样式（含移动端适配）
├── audio.js            # 音效系统
├── sim-node.js         # Node.js 平衡模拟器
├── js/
│   ├── pets.js         # 27只龟的数据定义（属性、被动、技能）
│   ├── engine.js       # 战斗引擎（回合系统、伤害计算、buff/debuff）
│   ├── skills.js       # 技能实现函数
│   ├── ui.js           # UI渲染（卡片、血条、状态图标、浮动数字）
│   ├── main.js         # 主流程（模式选择、选龟、结算、龟币）
│   └── tutorial.js     # 新手引导系统
├── assets/             # 59个图标/精灵图（9.2MB）
├── designs/            # 27份龟设计文档
└── docs/
    ├── 模块总结.md     # 本文档
    └── 后端需求.md     # 后端API需求
```

## 游戏模式

| 模式 | 说明 |
|------|------|
| **野生对局 (PVE)** | 2v2 对抗AI控制的随机龟队 |
| **Boss挑战** | 2v1 挑战增强Boss（HP×2.5, ATK×1.1, DEF/MR×1.3, 每回合2次行动） |
| **线上对战 (PVP)** | P2P实时对战，50龟币门票，赢家通吃100龟币 |

## 核心机制

### 伤害系统
- **物理伤害**（红色）：受护甲减免，公式 `减免% = DEF/(DEF+40)`
- **魔法伤害**（蓝色）：受魔抗减免，公式 `减免% = MR/(MR+40)`
- **真实伤害**（白色）：无视所有防御
- **护盾吸收**：泡泡护盾 → 普通护盾 → HP

### 增益/减益系统
- **数值型buff**（基于施法者属性）：攻击力+、护甲+、魔抗+
- **效果型debuff**（基于目标属性）：攻击力-%、护甲-%、魔抗-%
- **持续效果**：灼烧(40%ATK+8%HP魔法/回合)、诅咒(9%目标HP真实/回合)、治疗削减(-50%)
- **特殊状态**：闪避、恐惧、眩晕、虫洞标记、墨迹标记

### 特殊机制
| 机制 | 龟 | 说明 |
|------|-----|------|
| 墨迹增伤 | 线条龟 | applyRawDmg内全局生效，每层+5%受伤 |
| 怒气变身 | 熔岩龟 | 造伤+承伤转怒气，满100变身火山龟6回合 |
| 星能充放 | 星际龟 | 造伤62%转星能，每次技能后释放30%，满能流星100% |
| 装备系统 | 宝箱龟 | 累积财宝值抽装备，3池15件 |
| 随从系统 | 缩头乌龟 | 召唤A级以下龟，回合末自动行动，主死从亡 |
| 多重打击 | 赌神龟 | 每段40%概率额外打击，可连锁递减 |
| 结晶引爆 | 水晶龟 | 叠4层引爆10%目标HP魔法伤害 |
| 电击引爆 | 闪电龟 | 叠8层引爆82%ATK真实伤害 |
| 形态切换 | 双头龟 | 远程/近战切换，属性实时变化 |
| 涅槃重生 | 凤凰龟 | 首次死亡以30%HP复活 |
| 亡灵锁血 | 无头龟 | 首次濒死锁血1HP 2回合 |
| 技能复制 | 龟壳 | 随机复制敌方2技能以60%效果释放 |

## 龟币经济

| 场景 | 获得 |
|------|------|
| 每日首胜（PVE） | +50 龟币 |
| 每日首胜（Boss） | +50 龟币 |
| 普通胜利 | +10 + 回合数 龟币 |
| 普通失败 | +5 龟币 |
| 线上对战门票 | -50 龟币 |
| 线上对战胜利 | +100 龟币（赢家通吃） |
| 线上对战失败 | -50 龟币（门票不退） |

## 回合流程

```
beginTurn()
  ├── 减少CD、处理回合被动（石墙+DEF、竹编充能、糖果偷HP等）
  ├── 处理随从回合被动
  ├── 处理buff/debuff（灼烧DOT、治疗减、buff倒计时）
  └── 进入行动阶段

nextSideAction()
  ├── 左方行动（玩家/AI）× N只龟
  │   ├── 40秒计时器（超时自动出招）
  │   ├── 选龟 → 选技能 → 选目标
  │   └── executeAction() → 技能效果 → triggerOnHitEffects()
  ├── 右方行动（AI/对手）× N只龟
  └── finishSide()

finishSide() (双方行动完)
  ├── 随从回合末自动出招
  ├── 财神龟金币发放
  ├── 闪电龟回合电击
  ├── 龟壳储能波击
  ├── turnNum++ → beginTurn()
  └── 检查胜负
```

## 模拟器

`sim-node.js` — Node.js全引擎平衡模拟器

```bash
# 混合模拟（27龟 × 10搭档 × 5对手 = 1350场）
node sim-node.js --mixed

# 自定义对局
node sim-node.js 10 rainbow lightning lava crystal
```

- 加载真实游戏代码（pets.js + engine.js + skills.js + ui.js + main.js）
- Mock DOM/浏览器API
- 智能AI（每只龟有专属AI逻辑）
- 排除缩头乌龟（随从机制太复杂）

### 最新平衡数据（27只龟胜率 41%~62%）
