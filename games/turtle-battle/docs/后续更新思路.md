# 龟龟对战 — 后续更新思路

> 当前版本：**v2.0**（已定版，分支已建）
>
> 定位：3v3回合制策略战，作为龟投主站的社交娱乐入口

---

## v2.0 现状盘点

### 核心玩法 ✅
- 3v3回合制，前后排站位（6格阵型 + 嘲讽/前排优先/换位机制）
- 28只龟，120+技能，所有龟强度持平（稀有度≠战力）
- 等级系统（1-10级，每级+5%属性，Lv.4/7解锁技能索引3/4）
- 图鉴系统（书签式，可调等级+配置技能）
- 战场规则（8种每日随机：烈焰/铁壁/亡灵/深海/治愈/狂暴/雷暴/装备日）
- 12+组合技（双龟联动，需双方Lv.9+）

### 模式 ✅
- PVE 3v3（难度三档）
- Boss挑战 3v1（HP×3.5，每回合2行动）
- **深海闯关**（5关连续，血量延续，关间可重新站位，18件装备）
- P2P线上对战（PeerJS + TURN）

### 系统 ✅
- Web Audio API 音效系统（7个真实音效，低延迟 <5ms）
- BGM 系统（菜单/战斗/Boss 三首）
- 浮动数字系统（命中同步音效，伤害类型分色，堆叠避免重叠）
- 体型变化动画（最大HP变化时龟的精灵图 0.9x~1.15x 缩放）
- 龟币经济（每日首胜+对战奖励）
- 移动端适配（100dvh + touchmove锁定 + sheet面板）
- 新手引导

### 工程 ✅
- engine.js 拆分为 7 个文件（engine/fighter/combat/state/turn/action/ai）
- Playwright 自动化测试（119/120 技能通过）
- 确定性 RNG 同步（PVP）
- localStorage 持久化（等级/技能配置/龟币/音效静音）

## 优先级矩阵

| 紧急 × 重要 | 重要不紧急 | 紧急不重要 | 不紧急不重要 |
|-------------|-----------|-----------|------------|
| Bug修复 | 后端接入 | brief/数值一致性 | 语音录制 |
| 平衡调整 | 快速匹配 | 更多战场规则 | 3D场景 |

**原则**：先让 v2.0 稳定，再做后端联网；视觉升级等玩法验证完再投钱。

## AI + 0外包能做到什么

**AI 能做好**：
- 龟/UI/装备/背景 像素图（Midjourney/DALL-E）
- 所有代码逻辑（Claude 写）
- 概念设计/配色方案

**AI 做不好，需要手动/外包**：
- 多帧一致性动画（AI难以保证角色在不同帧的一致性）
- 高质量音效/BGM（版权问题）
- 统一UI精确像素对齐

**结论**：v2.0 → v2.3 都可以零成本。v3.0 美术升级阶段需要投入 ¥6000-16000 外包。
