# 缩头乌龟 (SS级) — 技能设计

## 核心定位
召唤师/坦克混合型。通过召唤一只随从龟获得额外行动力，自身偏防御向。

---

## 属性
| 属性 | 值 |
|------|-----|
| HP | 426 |
| ATK | 39 |
| DEF | 25 |
| MR | 24 |
| SPD | 6 |
| 暴击 | 25% |

---

## 被动：喊龟

- **开局时**随机召唤一只 **A级以下**的龟作为随从
- 随从拥有原龟 **40% 最大生命值**，其余属性（ATK/DEF/暴击等）正常
- 随从**不占队伍槽位**，作为缩头乌龟的附属存在
- 随从每**回合末**自动释放一个技能（有独立CD和AI）
- 随从躲在缩头乌龟身后，敌方单体技能无法选中随从，但会受到AOE伤害和敌方回合被动伤害
- **缩头乌龟阵亡时，随从也会一同阵亡**
- 随从阵亡会触发自身的死亡被动
- 随从的**被动技能正常生效**（如召唤到闪电龟则有电击被动，石头龟有反伤等）
- 随从的 **ATK/DEF/暴击** 等属性完全正常，**只有 HP 缩减为 40%**

### 召唤逻辑
1. 从 ALL_PETS 中筛选 rarity 为 C/B/A 的龟
2. 随机选一只（排除当前队伍已有的龟）
3. 创建完整 fighter（被动正常应用），仅 HP 和 maxHp 缩减为 40%
4. 放入缩头乌龟同侧队伍
5. 随从有独立的 HP 条、技能 CD，但不会在正常行动顺序中行动

---

## 一技能：攻击

- 造成 **100%×ATK** 物理伤害
- 缩头乌龟自身获得 **防御+20%** 持续 2 回合
- CD: 0

### 实现要点
- `physical` 类型 + `atkScale:1.0` + `selfDefUpPct:{pct:20, turns:2}`

---

## 二技能：防御

- 获得 **20% 最大HP** 的护盾，持续 **4回合**
- 4回合后护盾到期时，将**剩余护盾值的 20%** 转化为 HP 回复
- CD: 3

### 实现要点
- 使用 buff 类型 `hidingShield`，带 `turns` 和原始护盾值
- 在 `processBuffs` 中 tick：到期时检查 `shield` 剩余，回复 20%

---

## 三技能：指挥

- 命令被召唤的龟**立即释放 1次技能**（本回合随从共出手2次）
- 如果随从已阵亡，该技能无效（显示提示）
- CD: 3

### 实现要点
- 检查随从是否存活
- 从随从的 skills 中筛选可用技能
- 随机选1个，自动选择目标执行
- 类似自动 AI 出招逻辑

---

## 交互流程

```
缩头乌龟选择技能 ->
  一技能(攻击) / 二技能(防御):
    执行技能 -> 回合末随从自动放1个技能 -> 回合结束
  三技能(指挥):
    随从立即放1次技能 -> 回合末随从再自动放1个技能 -> 回合结束
    (随从一回合内共释放2次技能)
```

## 平衡考量

- 随从 40% HP 意味着大概 120~140 HP，很脆，容易被 AOE 清掉
- 缩头乌龟自身攻击力低（39 ATK），主要靠随从输出
- 如果随从是猎人龟/闪电龟等强力龟，会有很高的额外价值 -> 随机性是平衡点
- 如果随从是石头龟/小龟等弱龟，价值较低但至少能挡一次攻击
- 三技能"指挥"让随从额外释放1次技能，配合回合末自动出招 = 一回合随从出2次招
- 缩头乌龟阵亡时随从也会阵亡，保护缩头乌龟等于保护双人组
- HP426 + DEF25 + MR24 极肉，不容易被击杀
