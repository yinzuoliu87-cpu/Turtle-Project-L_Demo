# 龟壳 (SSS级) — 技能设计

## 核心定位
气场型重装。只有两个技能但拥有极强双被动：延迟自增益 + 储能释放。前6回合低调蓄力，6回合后全面强化；受到的伤害不会浪费，积攒后定期以波击倾泻。

---

## 被动一：气场觉醒

- **登场第6回合结束时**触发，获得以下永久增益（持续到战斗结束）：
  - **+10% ATK**（基于 baseAtk）
  - **+10% DEF**（基于 baseDef）
  - **+10% 最大生命值**（基于 baseHp），当前 HP 按比例同步提升
  - **10% 生命偷取**（所有造成的伤害均生效）
  - **10% 反伤**（受到伤害时反弹 10% 给攻击者）
  - **10% 穿甲**（基于 baseDef 的穿甲值，减少目标等效防御）
- 触发时日志 + 浮动文字提示觉醒

### 实现要点
- 新被动类型 `auraAwaken`
- 在回合计数（turnNum）达到 6 时，对拥有此被动的角色触发一次性增益
- ATK/DEF 增益：push `atkUp` / `defUp` buff（turns: 9999 或特殊标记永久）
- 最大生命值：`maxHp += Math.round(baseHp * 0.1)`，当前 HP 按比例 `hp = Math.round(hp * newMaxHp / oldMaxHp)`
- 生命偷取：设置 `f._auraLifesteal = 0.10`，在 `triggerOnHitEffects` 中检测并回血
- 反伤：设置 `f._auraReflect = 0.10`，在受伤逻辑中对攻击者造成 `Math.round(dmg * 0.10)` 伤害
- 穿甲：设置 `f.armorPen += Math.round(baseDef * 0.10)`
- 只触发一次，用标记 `f._auraAwakened = true` 防止重复

## 被动二：储能波击

- 受到的**所有伤害**会被储存为**能量值**（UI 显示能量条）
- 每 **9回合**自动释放一次**波击**（不占行动回合，在回合开始时触发）：
  - 对所有敌人造成 **储存值 × 0.008 × ATK** 的**普通伤害**
  - 自身获得 **储存值 × 0.01 × ATK** 的**护盾**，持续 **3回合**
  - 释放后清空储存值
- 能量条显示在 HP 条下方，颜色区分（如橙色）

### 实现要点
- 在被动对象中增加 `energyStore` 子类型
- `f._storedEnergy = 0`，每次受伤时 `f._storedEnergy += actualDmg`
- UI：在 fighter 卡片中增加能量条 `<div class="energy-bar">` ，宽度 = `storedEnergy / someCap * 100%`（无上限，但条最大显示到某阈值后用数字标注）
- 每9回合检测（turnNum % 9 === 0），对所有拥有此被动的角色触发波击：
  - 伤害 = `Math.round(f._storedEnergy * 0.008 * f.atk)`
  - 护盾 = `Math.round(f._storedEnergy * 0.01 * f.atk)`，push `shield` buff（turns:3）
  - 清空 `f._storedEnergy = 0`
- 日志显示储存值、造成伤害、获得护盾

## 一技能：攻击

- 对主目标造成 **6段伤害**，普通与穿透**交替**：
  - 第1段：普通（0.1×ATK）
  - 第2段：穿透（0.1×ATK）
  - 第3段：普通（0.1×ATK）
  - 第4段：穿透（0.1×ATK）
  - 第5段：普通（0.1×ATK）
  - 第6段：穿透（0.1×ATK）
  - 共 **0.6×ATK 普通 + 0.6×ATK 穿透 = 1.2×ATK 总伤害**
- 对**次目标**（其他所有存活敌人）造成此次总伤害的 **10%** 溅射伤害
- CD: 0

### 实现要点
```javascript
{ name:'攻击', type:'shellStrike', hits:6, power:0, pierce:0,
  desc:'6段交替普通/穿透共1.2×ATK，溅射10%',
  cd:0, totalScale:1.2, splashPct:10 }
```
- 新技能类型 `shellStrike`
- 类似寒冰龟 `iceSpike` 的交替逻辑：奇数段普通伤害（受防御），偶数段穿透伤害（无视防御）
- 每段 0.2×ATK
- 主目标命中后，对其他存活敌人造成 `Math.round(totalDmgDealt * 0.10)` 溅射普通伤害
- 溅射伤害作为单独一次伤害结算，显示浮动数字

## 二技能：复制

- **随机选取**一个存活敌人
- **随机复制**该敌人的一个技能并释放
- 技能中所有数值加成（ATK倍率、护盾值等）**基于龟壳自身属性**计算
- CD: 4

### 实现要点
```javascript
{ name:'复制', type:'shellCopy', hits:0, power:0, pierce:0,
  desc:'随机复制敌方一个技能并释放',
  cd:4 }
```
- 新技能类型 `shellCopy`
- 实现步骤：
  1. 从存活敌人中随机选一个
  2. 从该敌人的 skills 中随机选一个（排除 `shellCopy` 自身防递归）
  3. 以龟壳为 attacker 执行该技能（调用对应的 `executeAction` 逻辑）
  4. 目标选择：单体技能选血量最低敌人，AOE/多段随机技能按原逻辑
- 日志显示复制了谁的什么技能
- 注意：复制的技能不消耗原技能的CD，龟壳只消耗自己的CD

## 伤害模拟（龟壳 ATK=42，DEF=18，觉醒后 ATK≈46）

### 觉醒前 vs 觉醒后属性
| 属性 | 觉醒前 | 觉醒后 |
|------|--------|--------|
| ATK | 42 | 46 |
| DEF | 18 | 20 |
| MaxHP | 360 | 396 |
| 生命偷取 | 0% | 10% |
| 反伤 | 0% | 10% |
| 穿甲 | 0 | 2 |

### 一技能：攻击（觉醒后，对DEF=15目标）
- 普通段：0.1×46≈5 → 减防后≈3
- 穿透段：0.1×46≈5（无视防御）
- 6段总伤 ≈ 3+5+3+5+3+5 = 24
- 溅射：24×10% ≈ 2（对其他敌人）

### 储能波击（假设储存500能量，觉醒后ATK=46）
- 波击伤害 = 500 × 0.008 × 46 = 184（对全体敌人）
- 护盾 = 500 × 0.01 × 46 = 230（持续3回合）

## 平衡考量

- **前6回合脆弱期**：觉醒前只是普通属性的SSS级，没有任何被动加成，容易被集火击杀
- **觉醒后全面强化**：+10%全属性 + 偷取/反伤/穿甲，成为持久战怪物
- **储能机制风险与收益**：受伤越多波击越强，但也意味着要扛住伤害才能释放
- **只有两个技能**：技能组简单但被动极强，操作感偏向"站桩输出"
- **复制技能不确定性**：可能复制到强力技能也可能复制到辅助技能，增加博弈感
- **反伤+偷取组合**：鼓励敌方控制输出节奏，避免无脑集火龟壳
