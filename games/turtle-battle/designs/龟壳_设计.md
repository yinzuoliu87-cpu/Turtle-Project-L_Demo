# 龟壳 (SSS级) — 技能设计

## 核心定位
气场型重装。只有两个技能但拥有极强双被动：延迟自增益 + 储能释放。前4回合低调蓄力，4回合后全面强化；受到的伤害不会浪费，积攒后定期以波击倾泻。

## 被动一：气场觉醒

- **登场第4回合结束时**触发，获得以下永久增益（持续到战斗结束）：
  - **+12% ATK**（基于 baseAtk）
  - **+12% DEF**（基于 baseDef）
  - **+12% 最大生命值**（基于 baseHp），当前 HP 按比例同步提升
  - **12% 生命偷取**（所有造成的伤害均生效）
  - **12% 反伤**（受到伤害时反弹 12% 给攻击者）
  - **12% 穿甲**（基于 baseDef 的穿甲值，减少目标等效防御）
- 触发时日志 + 浮动文字提示觉醒

### 实现要点
- 被动类型 `auraAwaken`
- 在回合计数（turnNum）达到 4 时，对拥有此被动的角色触发一次性增益
- ATK/DEF 增益：push `atkUp` / `defUp` buff（turns: 9999 或特殊标记永久）
- 最大生命值：`maxHp += Math.round(baseHp * 0.12)`，当前 HP 按比例 `hp = Math.round(hp * newMaxHp / oldMaxHp)`
- 生命偷取：设置 `f._auraLifesteal = 0.12`，在 `triggerOnHitEffects` 中检测并回血
- 反伤：设置 `f._auraReflect = 0.12`，在受伤逻辑中对攻击者造成 `Math.round(dmg * 0.12)` 伤害
- 穿甲：设置 `f.armorPen += Math.round(baseDef * 0.12)`
- 只触发一次，用标记 `f._auraAwakened = true` 防止重复

## 被动二：储能波击

- 受到的**所有伤害**会被储存为**能量值**（UI 显示能量条）
- 每 **4回合**自动释放一次**波击**（不占行动回合，在回合开始时触发）：
  - 对所有敌人造成 **储存值 x 0.008 x ATK** 的**普通伤害**
  - 自身获得 **储存值 x 0.01 x ATK** 的**护盾**，持续 **3回合**
  - 释放后清空储存值
- 能量条显示在 HP 条下方，颜色区分（如橙色）

### 实现要点
- 在被动对象中增加 `energyStore` 子类型
- `f._storedEnergy = 0`，每次受伤时 `f._storedEnergy += actualDmg`
- UI：在 fighter 卡片中增加能量条
- 每4回合检测（turnNum % 4 === 0），对所有拥有此被动的角色触发波击：
  - 伤害 = `Math.round(f._storedEnergy * 0.008 * f.atk)`
  - 护盾 = `Math.round(f._storedEnergy * 0.01 * f.atk)`，push `shield` buff（turns:3）
  - 清空 `f._storedEnergy = 0`

## 一技能：攻击

- 对主目标造成 **6段伤害**，物理与真实**交替**：
  - 第1段：物理（20%xATK）
  - 第2段：真实（20%xATK）
  - 第3段：物理（20%xATK）
  - 第4段：真实（20%xATK）
  - 第5段：物理（20%xATK）
  - 第6段：真实（20%xATK）
  - 共 **60%xATK 物理 + 60%xATK 真实 = 120%xATK 总伤害**
- 对**次目标**（其他所有存活敌人）造成此次总伤害的 **25%** 溅射伤害
- CD: 0

### 实现要点
```javascript
{ name:'攻击', type:'shellStrike', hits:6, power:0, pierce:0,
  desc:'6段物理/真实交替共120%ATK，溅射25%',
  cd:0, totalScale:1.2, splashPct:25 }
```
- 技能类型 `shellStrike`
- 奇数段物理伤害（受DEF），偶数段真实伤害（无视DEF和MR）
- 每段 20%xATK
- 主目标命中后，对其他存活敌人造成 `Math.round(totalDmgDealt * 0.25)` 溅射伤害

## 二技能：复制

- **随机选取**一个存活敌人
- **随机复制**该敌人的 **2个技能**，以 **60%效果** 释放
- CD: 4

### 复制黑名单
以下技能类型不可被复制（防止递归/逻辑冲突）：
`shellCopy`, `twoHeadSteal`, `cyberDeploy`, `cyberBuff`, `hidingDefend`, `hidingCommand`, `diceFate`, `fortuneDice`, `fortuneAllIn`, `bambooHeal`, `bambooLeaf`, `ghostPhase`, `diamondFortify`, `iceShield`, `twoHeadSwitch`, `mechAttack`, `chestOpen`, `gamblerDraw`, `gamblerBet`, `chestCount`, `chestSmash`, `starWormhole`

### 实现要点
```javascript
{ name:'复制', type:'shellCopy', hits:0, power:0, pierce:0,
  desc:'随机复制敌方2技能60%效果释放',
  cd:4 }
```
- 技能类型 `shellCopy`
- 实现步骤：
  1. 从存活敌人中随机选
  2. 从该敌人的 skills 中随机选2个（排除黑名单中的技能类型）
  3. 以龟壳为 attacker 执行这2个技能，所有数值x60%
  4. 目标选择：单体技能选血量最低敌人，AOE/多段随机技能按原逻辑
- 日志显示复制了谁的什么技能
- 注意：复制的技能不消耗原技能的CD，龟壳只消耗自己的CD

## 平衡考量

- **前4回合蓄力期**：觉醒前只是普通属性的SSS级，没有被动加成，容易被集火击杀
- **觉醒后全面强化**：+12%全属性 + 偷取/反伤/穿甲，成为持久战怪物
- **储能机制风险与收益**：受伤越多波击越强，但也意味着要扛住伤害才能释放
- **只有两个技能**：技能组简单但被动极强，操作感偏向"站桩输出"
- **复制2技能60%效果**：可能复制到强力技能组合，增加博弈感和不确定性
- **25%溅射**：一技能自带AOE能力，对多目标场景输出稳定
- **反伤+偷取组合**：鼓励敌方控制输出节奏，避免无脑集火龟壳
